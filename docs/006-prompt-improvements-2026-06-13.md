## 问题 1：算术幻觉穿透仲裁层 ✅ 已修复

### 问题描述

**现象**：
- 激进分析师声称"SCFI+45% 而股价仅涨 1%，41 倍运价涨幅只兑现 1% 股价涨幅"
- 事实仲裁员核验了 SCFI 绝对涨幅（+45%），但**未拆解验证**：
  - 股价在同期的实际涨跌幅
  - "45% ÷ 1% = 41 倍"的算术是否成立

**根本原因**：
提示词要求"全局最多抽查 5 个派生数值"，但**跨资产相对倍数**（如"A 涨 X% vs B 涨 Y% = Z 倍"）未被列入必查清单。

### 修复方案

**文件**: `backend/app/ai/llm_engine/prompts/templates.py`  
**位置**: `SYSTEM_PROMPT_FACT_ARBITRATION_CN` 的"数值仲裁规则"第 8 条（新增）

**新增规则**：
```markdown
8. **跨资产相对倍数必须拆解验证**：
   若任一 Agent 使用"A 涨 X% 而 B 仅涨 Y%，相差 Z 倍"或"A 涨幅是 B 的 Z 倍"类叙述，必须拆解验证：
   - 先确认 A 和 B 的起止时间是否一致（同一时间窗口）
   - 再分别核验 A 和 B 的涨跌幅（调用 `query_and_calculate` 或 `execute_python_sandboxed` 获取原始时间序列）
   - 最后核验 X/Y 是否等于 Z，或 X 是否为 Y 的 Z 倍
   禁止直接采信"某某倍"的叙述性倍数，必须回溯原始时间序列。
   示例："SCFI+45% vs 股价+1% = 41 倍"需拆解为：
   ① SCFI 在 [起始日, 结束日] 的涨幅 = ?（需提供具体日期和数值）
   ② 股价在 [起始日, 结束日] 的涨跌幅 = ?（需提供具体日期和数值）
   ③ 核验 45% ÷ 1% 是否等于 41 倍，或重新计算实际倍数关系。
```

**预期效果**：
- 事实仲裁员在看到"SCFI+45% vs 股价+1%"时，会自动：
  1. 调用工具获取股价在同期（4月底-6月5日）的实际涨跌幅
  2. 核验 45% ÷ 股价涨幅 是否等于 41 倍
  3. 若不等，列入"数值核验"表并纠正

## 问题 2：纸上止损 ❌ 不存在

### 验证结果

经代码审查，系统**已有完整的持仓止损止盈扫描机制**：

1. **PM 决策输出** → `stop_loss` / `take_profit` / `holding_horizon_days`
2. **写入持仓表** → `pm_rules.py::sync_pm_discipline_to_position()`
3. **定时扫描** → `position_discipline_scheduler.py` 每隔 N 秒自动扫描
4. **触发判定** → `pm_rules.py::evaluate_position_disciplines()` 纯确定性比较
5. **启动复议** → `discipline_service.py::_handle_position_discipline_trigger()` 自动创建辩论

**结论**：PM 输出的 `stop_loss=14.00` 会被系统自动监控并触发，不是"纸上止损"。

## 优化 2：PM 决策事项优先级分级 ✅ 已实施

### 改进前

```markdown
## 决策简报
| 项目 | 内容 |
|------|------|
| **需 PM 决策事项** | ①止损14.00是否维持（建议维持）；②若反弹至14.50附近缩量滞涨是否主动减仓（本轮不做强制指令，留给PM自主判断）；③SCFI下周数据若回落是否提前复议 |
```

### 改进后

```markdown
## 决策简报

| 项目 | 内容 |
|------|------|
| **signal** | [买入/持有/卖出] |
| **confidence** | [0-100，说明主要加分项和扣分项] |
| **当前仓位** | [N股（X%）] |
| **目标仓位** | [N股（Y%）] |
| **仓位变化** | [维持/增持X%/减持X%/清仓] |
| **最关键证据** | [最能改变仓位、置信度、止损/止盈的1-2条核心证据] |
| **最大反证** | [最可能推翻当前决策的后续证据或事件] |
| **交易影响** | [立即执行/分批建仓/等待确认/维持观望/部分止盈] |
| **需PM决策事项** | 见下表 |

**需PM决策事项优先级**

| 优先级 | 决策事项 | 触发条件 | 默认处理 |
|-------|---------|---------|---------|
| P0（必须立即决策） | [例：止损14.00是否维持] | [例：价格跌破14.00] | [例：建议维持，跌破则清仓] |
| P1（重要，需关注） | [例：SCFI下周数据若回落] | [例：SCFI连续2周<2200] | [例：提前复议，可能减仓] |
| P2（可选，PM自主） | [例：反弹至14.50是否减仓] | [例：价格至14.50且缩量] | [例：留给PM自主判断] |
```

**影响范围**：
- `SYSTEM_PROMPT_PORTFOLIO_MANAGER_CN`（PM 决策报告格式）

**好处**：
1. **优先级清晰**：P0/P1/P2 三级，明确哪些必须决策、哪些可选
2. **结构化呈现**：决策事项、触发条件、默认处理分栏展示
3. **减少认知负担**：用户可优先关注 P0 事项
4. **便于追溯**：清晰记录每个决策点的条件和处理方式

## 待实施改进（P1）

### 问题 3：记忆召回超时

**现象**：PM 报告写明"记忆召回超时，未获取到相关经验"

**建议方案**：

在 PM 提示词中新增"记忆召回失败降级策略"：

```markdown
## 【记忆召回失败降级策略】
若 `recall_memory` 超时、失败或无结果：
1. **优先使用 `previous_pm_decision` 和 `same_stock_history`**：
   - `previous_pm_decision` 包含上一轮同股 PM 决策（decision/target_position/止损/止盈/置信度）
   - `same_stock_history` 包含历史订单、成交、已实现盈亏和历史 PM 决策摘要
   - 这两个字段是结构化输入，无需召回 Memory 即可获取
2. **从中提取替代经验**：
   - 上一轮 PM 决策的核心理由（为什么 buy/hold/sell）
   - 历史已实现盈亏（是否有亏损交易需要吸取教训）
   - 历史止损/止盈执行记录（哪些纪律有效、哪些失效）
3. **在报告中写明降级原因**：
   "记忆系统召回超时/失败，已从 previous_pm_decision 和 same_stock_history 提取替代经验：[具体内容]"
4. **不要因为 Memory 缺失就降低决策质量**：
   当前事实、工具结果、上一轮 PM 决策和历史交易记录已足以支撑高质量决策。
```

**状态**：待用户确认后实施
