# 仓库股票自动模拟盘轻量增强设计

## 背景

当前系统已经具备模拟账户、订单、成交、持仓和交易记录能力，但缺少一个持续记录账户净值的口径，也缺少一个简单接口回答“模拟盘是否跑赢基准”。因此首版只补齐绩效可见性，不调整交易流程。

本设计只做两件事：

- 增加账户每日快照。
- 增加简单绩效查询。

## 目标

- 每个交易日收盘后记录一次模拟账户快照。
- 使用沪深 300 作为默认基准，记录账户与基准的同期表现。
- 提供简单 API 查询账户净值、累计收益、超额收益和最大回撤。
- 不改变现有买卖股票流程。

## 非目标

- 不修改 `execute_trading_order`。
- 不新增交易工具限制。
- 不新增前端风控配置。
- 不优化仓库股票自动分析调度。
- 不增强经验复盘上下文。
- 不新增交易审计表。
- 不做复杂组合管理、组合优化、行业归因或持仓贡献分析。
- 不接真实券商。
- 不做盘中高频、盘口撮合或历史回测。

## 现有能力复用

- `backend/app/models/account.py`：已有账户总资产、现金、市值、盈亏和交易统计字段。
- `backend/app/models/position.py`：已有当前持仓、成本、市值和浮盈亏字段。
- `backend/app/models/trade_record.py`：已有成交记录、费用和成交时间。
- `backend/app/models/data_storage.py`：已有指数日线数据模型，可读取沪深 300 日线作为基准。
- `backend/app/tasks/async_scheduler.py`：已有异步任务调度框架，可挂载收盘后快照任务。

## 功能范围

### 1. 增加账户每日快照

新增一个轻量任务，在交易日收盘后记录每个用户模拟账户的表现。即使当天没有交易，也应记录快照，保证净值曲线连续。

快照记录内容：

- 用户 ID。
- 账户 ID。
- 快照日期。
- 总资产。
- 可用现金。
- 持仓市值。
- 持仓数量。
- 当日收益率。
- 累计收益率。
- 基准代码，默认 `000300.SH`。
- 基准当日收益率。
- 基准累计收益率。
- 相对基准超额收益。
- 截至当日最大回撤。

收益计算口径：

- 账户当日收益率 = 当日总资产 / 上一个快照总资产 - 1。
- 账户累计收益率 = 当日总资产 / 初始基准资产 - 1。
- 基准当日收益率 = 当日基准收盘价 / 上一个基准收盘价 - 1。
- 基准累计收益率 = 当日基准收盘价 / 起始基准收盘价 - 1。
- 超额收益 = 账户累计收益率 - 基准累计收益率。
- 最大回撤 = 从账户历史净值高点到当前净值的最大跌幅。

首个快照处理：

- 首个账户快照的当日收益率记为 `0`。
- 首个基准快照的当日收益率记为 `0`。
- 初始基准资产使用首个快照的账户总资产。
- 起始基准价格使用首个快照日期对应或之前最近一个交易日的基准收盘价。

基准缺失处理：

- 若当日没有沪深 300 数据，使用快照日期之前最近一个交易日的指数收盘价。
- 若找不到任何可用基准价格，仍记录账户快照，但基准收益和超额收益置空，并记录缺失原因。

### 2. 增加简单绩效查询

新增或扩展 API，提供模拟盘绩效查询能力。

首版查询内容：

- 当前总资产。
- 当前可用现金。
- 当前持仓市值。
- 当前持仓数量。
- 累计收益率。
- 沪深 300 同期收益率。
- 超额收益。
- 最大回撤。
- 总交易次数。
- 最近快照日期。
- 净值曲线。
- 基准对比曲线。

建议 API：

- `GET /api/v1/performance/summary`：查询当前用户模拟账户绩效摘要。
- `GET /api/v1/performance/equity-curve`：查询当前用户模拟账户净值曲线和基准曲线。

查询参数：

- `start_date`：可选，查询开始日期。
- `end_date`：可选，查询结束日期。
- `benchmark_code`：可选，首版默认只支持 `000300.SH`。

首版不提供：

- 夏普比率。
- 信息比率。
- 行业归因。
- 单股贡献。
- 持仓归因。
- 导出报表。

## 数据模型建议

### 新增 `account_equity_snapshots`

新增账户净值快照表，用于长期绩效评测。

核心字段：

- `snapshot_id`：快照 ID。
- `user_id`：用户 ID。
- `account_id`：账户 ID。
- `snapshot_date`：快照日期。
- `total_assets`：总资产。
- `available_cash`：可用现金。
- `market_value`：持仓市值。
- `position_count`：持仓数量。
- `daily_return`：当日收益率。
- `cumulative_return`：累计收益率。
- `benchmark_code`：基准代码。
- `benchmark_close`：基准收盘价。
- `benchmark_daily_return`：基准当日收益率。
- `benchmark_cumulative_return`：基准累计收益率。
- `excess_return`：相对基准超额收益。
- `max_drawdown`：截至当日最大回撤。
- `missing_benchmark_reason`：基准数据缺失原因。
- `created_at`：创建时间。

唯一约束：

- 同一 `account_id + snapshot_date + benchmark_code` 只能有一条快照。

## 调度设计

快照任务接入现有异步调度框架。

建议默认规则：

- 交易日收盘后运行。
- 每天只生成一次同日期快照。
- 如果重复执行同一天任务，使用更新或跳过策略，避免重复快照。
- 如果当天不是交易日，可以跳过。
- 如果需要补数据，允许按日期手动重跑快照任务。

## 实施步骤

### 第一步：增加快照模型

- 新增 `AccountEquitySnapshot` 模型。
- 注册模型导入，确保建表流程能识别。
- 增加必要索引和唯一约束。

### 第二步：实现快照计算服务

- 读取当前用户账户。
- 统计当前持仓数量。
- 读取最近一条账户快照。
- 读取沪深 300 最近可用收盘价。
- 计算当日收益、累计收益、基准收益、超额收益和最大回撤。
- 写入或更新当天快照。

### 第三步：接入收盘后任务

- 在任务调度注册表中增加账户快照任务。
- 默认在交易日收盘后运行。
- 任务失败时记录日志，不影响现有交易和分析流程。

### 第四步：增加绩效查询 API

- 增加绩效摘要接口。
- 增加净值曲线接口。
- 支持按日期区间查询。
- 首版默认基准为 `000300.SH`。

## 测试策略

### 单元测试

- 首个快照收益率计算正确。
- 后续快照当日收益率计算正确。
- 累计收益率计算正确。
- 基准收益率计算正确。
- 超额收益计算正确。
- 最大回撤计算正确。
- 基准缺失时仍能记录账户快照。

### 集成测试

- 快照任务能为账户生成当日快照。
- 重复执行同一天任务不会生成重复快照。
- 绩效摘要接口返回最新快照指标。
- 净值曲线接口返回账户曲线和基准曲线。

### 回归测试

- 现有交易工具行为不变。
- 现有手动下单 API 行为不变。
- 现有账户、持仓和交易记录查询不变。

## 风险与边界

- 该方案只提供绩效度量，不保证盈利。
- 如果账户总资产没有及时更新，快照会继承现有账户口径的误差。
- 如果基准指数数据缺失，超额收益会暂时不可用。
- 当前模拟交易仍缺少滑点、涨跌停、停牌、流动性和部分成交等真实约束，收益可能偏乐观。
