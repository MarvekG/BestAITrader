# CRUD 模块设计与约束

`crud` 提供数据库读写的薄封装。它负责常见查询和持久化动作，但不应成为隐藏业务流程的地方。

## 职责

- 封装重复的 SQLAlchemy 查询、创建、更新和统计逻辑。
- 为 service、workflow 和 endpoint 提供稳定的数据访问函数。
- 处理局部查询条件和排序分页。

## 设计约束

- CRUD 函数保持薄而明确，不写跨表业务状态机。
- 事务边界通常由 service 控制；CRUD 不应随意 commit 多个业务步骤。
- 查询必须绑定必要的用户或 session 条件，避免跨用户数据泄漏。
- 返回值语义要清楚：不存在、无权限、已删除和空结果不要混成同一种情况。
- 新增复杂查询时优先让调用方传入明确参数，不读取全局业务状态。

## 验证

- 运行相关 service/API 测试。
- 涉及用户作用域时补查鉴权和归属校验路径。
