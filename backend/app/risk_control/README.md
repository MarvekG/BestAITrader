# Risk Control 模块设计与约束

`risk_control` 负责组合和订单层面的风控评估。它给出是否允许、风险等级和原因，不直接执行订单，也不绕过交易服务。

## 职责

- 对拟下单请求进行组合层预检。
- 评估持仓集中度、现金、仓位、个股风险和规则阈值。
- 为手动交易和 AI 交易提供统一风控判断。

## 设计约束

- 订单风控统一复用 `portfolio_risk_control_service.evaluate_order`。
- 风控服务不能直接写订单或成交；执行仍由 `TradingService` 完成。
- 风控结果要可解释，前端和审计需要知道触发了哪些约束。
- 风控规则应基于账户、持仓、行情和配置，不根据调用来源给出不一致判断。
- 新增规则要考虑手动/API/AI 三条下单路径的一致性。

## 修改入口

- 风控服务：`service.py`
- 设计细节：`DESIGN.md`
