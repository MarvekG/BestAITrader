# Performance 模块设计与约束

`performance` 负责账户和组合表现统计。它读取交易账本、权益快照和行情数据，输出收益、回撤和绩效指标，不参与交易执行。

## 职责

- 计算账户收益、权益曲线、回撤和阶段表现。
- 为前端绩效页和组合概览提供指标。
- 与交易、组合估值共享同一套账户和持仓口径。

## 设计约束

- 绩效模块只读交易事实和快照，不直接修正订单、成交、持仓。
- 指标口径变更要保持前端展示和历史数据解释一致。
- 缺少行情或权益快照时应显式暴露数据不足，不输出看似精确的指标。
- 后验决策质量分析归 `ai/experience`，这里不新增平行复盘逻辑。

## 修改入口

- 绩效服务：`service.py`

## 验证

- 运行 performance、portfolio、trading 相邻测试。
