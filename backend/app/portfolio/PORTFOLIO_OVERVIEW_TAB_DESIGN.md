# 模拟交易组合概览 Tab 设计

## 背景

当前系统已经具备模拟账户、订单、成交、持仓、T+1 持仓账本和绩效快照能力。模拟交易页可以查看账户资产、持仓列表、订单记录和绩效表现，但缺少一个从组合视角回答“现金和持仓如何分布”“单股和行业是否集中”的轻量视图。

本设计在现有模拟交易中心中新增一个独立的“组合概览” Tab，只展示当前组合结构，不改变交易、撮合、绩效快照或 AI 决策流程。

## 目标

- 在模拟交易页新增与“绩效表现”并列的“组合概览” Tab。
- 展示现金比例、持仓比例、持仓总市值、持仓数量等基础组合摘要。
- 展示单股权重、行业分布和浮盈亏排行。
- 复用现有账户、持仓、实时行情和股票基础信息数据。
- 保持项目研究和模拟交易定位，不提供真实资金组合管理能力。

## 非目标

- 不新增数据库表。
- 不做调仓建议。
- 不做组合优化。
- 不新增风控规则或下单拦截。
- 不改变 `TradingService`、`TradingEngine`、订单、成交、持仓账本和绩效快照逻辑。
- 不接入真实券商或真实资金账户。
- 不新增行业归因、单股贡献归因、夏普比率或信息比率。

## 后端设计

新增轻量组合模块：

- `backend/app/portfolio/service.py`：负责组合概览聚合计算。
- `backend/app/api/endpoints/portfolio.py`：提供当前用户组合概览 API。

核心接口：

- `GET /api/v1/portfolio/overview`

数据来源：

- `Account`：账户现金、冻结资金、初始资产等基础字段。
- `Position`：有效持仓、股数、均价、持仓快照价、可卖股数、浮盈亏。
- `StockBasic`：股票名称和行业。
- `StockRealtimeMarket`：每只股票最新价格。

动态估值口径：

- 仅统计 `Position.total_shares > 0` 的有效持仓。
- 单股最新价优先使用最新 `StockRealtimeMarket.current_price`。
- 若实时行情缺失，则回退到 `Position.current_price`。
- 若持仓当前价缺失或无效，则回退到 `Position.avg_cost`。
- 单股市值 = 最新价 * 持仓股数。
- 动态总资产 = 可用现金 + 冻结现金 + 动态持仓市值。
- 空仓时持仓市值为 0，现金比例为 1。

返回结构：

- `summary`：总资产、可用现金、冻结现金、持仓市值、现金比例、持仓比例、持仓数量。
- `positions`：股票代码、名称、行业、股数、可卖数量、最新价、均价、市值、权重、浮盈亏、浮盈亏率。
- `industry_allocations`：行业、行业市值、行业权重、持仓数量、股票代码列表。
- `top_weights`：按单股权重降序排序的前 5 项。
- `top_gainers`：按浮盈亏金额降序排序的前 5 项。
- `top_losers`：按浮盈亏金额升序排序的前 5 项。

边界处理：

- 当前用户没有账户时，通过现有 `ensure_user_account` 创建默认模拟账户。
- 股票基础信息缺失时，股票名使用 `Unknown`，行业归为 `未知行业`。
- 动态总资产小于等于 0 时，所有权重返回 0，避免除零。
- 组合概览仅用于展示，不写入数据库。

## 前端设计

新增文件：

- `frontend/src/api/portfolio.ts`：定义组合概览类型和 API 调用。
- `frontend/src/features/trading/PortfolioOverviewTab.tsx`：组合概览 Tab 展示组件。

修改文件：

- `frontend/src/pages/SimulatedTradingPage.tsx`：新增“组合概览” Tab。
- `backend/app/locales/zh.json` 和 `backend/app/locales/en.json`：补充 Tab 与表格文案。

页面结构：

- 顶部统计卡：总资产、现金比例、持仓比例、持仓数量。
- 持仓权重表：按权重降序展示单股权重和浮动盈亏。
- 行业分布表：行业、市值、权重、持仓数量、股票列表。
- 浮盈亏排行：盈利排行和亏损排行两个表格。

响应式要求：

- 桌面端使用统计卡和并列表格，保持与现有绩效 Tab 的 Ant Design 风格一致。
- 移动端使用 `Row` / `Col` 自适应，表格开启横向滚动。

## 测试策略

后端测试：

- 有现金、两只持仓和实时行情时，接口返回动态总资产、现金比例、单股权重和行业分布。
- 缺少股票基础信息或实时行情时，接口使用回退价格和未知行业。
- 空仓账户返回空列表、现金比例 1、持仓比例 0。
- 未登录访问由现有认证依赖拦截。

前端验证：

- `npm run typecheck` 保证类型正确。
- `npm run build` 保证页面可编译。

## 风险与边界

- 实时行情缺失时，估值会使用持仓快照价或成本价，可能与真实市场价格有差异。
- 行业分布依赖股票基础信息，未同步基础信息时会归到未知行业。
- 本功能只展示模拟组合结构，不代表投资建议或真实资金组合管理。
